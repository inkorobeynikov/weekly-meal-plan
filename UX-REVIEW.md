# UX-ревью приложения — W01–W09

> Первый проход сделан по коду экранов (`apps/mobile/app`) и макету (`.design-ref`). Severity: **BLOCKER** (ломает поток) · **ANNOYING** (раздражает) · **COSMETIC** (мелочь).
> Под каждым пунктом — реши: `[ ] чинить` / `[ ] не важно`, и допиши свои ощущения. Это превратится в промпты для агента.

---

## ⭐ Приоритет (с чего начать)

1. **W09 Feedback недостижим из приложения** (BLOCKER). Экран есть, но войти в него можно только по тапу из push. Нет ни таба, ни кнопки после покупок. Петля «оценил неделю → улучшили план» оборвана.
2. **W08 «Новый тиждень» ломает поток** (BLOCKER). Кнопка дёргает генерацию и сразу уходит на план, ошибку глотает, фидбек пропускает. Юзер оказывается на пустом плане без объяснений.
3. **Картинок нет нигде** (ANNOYING, по всему приложению). `imageUri` не прокидывается план → карточка → рецепт, поэтому везде серые плейсхолдеры. Дизайн — фото-центричный, приложение — нет.
4. **«Интеллект» из макета вырезан** (ANNOYING, определяет продукт). Нет AI-объяснений «почему это блюдо», бейджей (Kid-ok, Leftovers, Try-new, цена), структурных опций замены, промо-подсказок в списке. Получился аккуратный, но generic список блюд — а продаёшь ты AI-планировщик.

## W06 — Онбординг (`app/onboarding/index.tsx`)

- **ANNOYING · BEHAVIOR:** Имя семьи (шаг 1) и размер семьи (шаг 2) собираются, но **не сохраняются** — `finish()` шлёт только аллергии/ограничения (TODO: бэкенд не умеет). Первые два шага — бутафория. → _[ ] чинить / [ ] не важно:_
- **ANNOYING · FLOW:** «Pomiń» есть на шаге 1 (прыгает на шаг 3), но нет на шаге 2 — непоследовательно. → _[ ]:_
- **COSMETIC · COPY:** Заголовок «Jak masz na imię?» («как тебя зовут?») не совпадает с полем «Nazwa rodziny» (имя семьи). → _[ ]:_
- **COSMETIC · BEHAVIOR:** Ошибка сохранения показывается, но экран сразу анмаунтится (`router.replace`) — юзер её не видит, dead code. → _[ ]:_
- Сильные стороны: прогресс-точки, добавление своих ограничений, проброс HARD-CONSTRAINT — ок.

## W01 — Недельный план (`(tabs)/plan/index.tsx`)

- **BLOCKER-ish · BEHAVIOR:** Поллинг генерации **без потолка** — если фоновая задача не завершится, экран опрашивает каждые 8с вечно, без «слишком долго / повторить». → _[ ]:_
- **ANNOYING · VISUAL:** `MealCard` без `imageUri` → всегда плейсхолдер; тап открывает рецепт тоже без картинки. → _[ ]:_
- **ANNOYING · FLOW:** С главного экрана плана **нельзя заменить блюдо** (swap только в Review/Detail). И нет пути к фидбеку. → _[ ]:_
- **ANNOYING · MISSING vs макет:** Нет AI-блока «почему такой план», полоски ограничений, бейджей блюд. → _[ ]:_
- Сильное: явные loading/ready/empty/error, pull-to-refresh, 404→empty, поддержка 7–14 дней.

## W04 — Ревью плана (`(tabs)/plan/review.tsx`)

## Сквозные темы

1. **Асинхронная генерация обрабатывается непоследовательно** — W01 поллит правильно, W04 и W08 «выстрелил и ушёл». Привести всё к паттерну W01. _(BLOCKER для W08, ANNOYING для W04.)_
2. **Картинок нет** — `imageUri` не прокидывается нигде. _(ANNOYING, по всему приложению.)_
3. **Дыры бэкенда оставляют мёртвый/исчезающий UI** — имя/размер семьи, добавление члена, цена товара, «новый тиждень». Либо гейтить, либо показывать ошибку. _(ANNOYING.)_
4. **Слой «интеллекта» из макета вырезан** — нет AI-объяснений, бейджей, структурных замен, промо в покупках, сворачиваемых секций рецепта. _(ANNOYING — определяет продукт.)_
5. **Недельная петля разорвана** — фидбек недостижим, approve не показывает список покупок, «новый тиждень» пропускает фидбек. _(BLOCKER.)_
6. **Дизейбл вместо скрытия** — мёртвая swap-CTA в библиотеке, пустой таб рецептов. _(COSMETIC.)_
7. **Польские тексты в целом хорошие** — лишь мелкие несостыковки (онбординг, «Średnio»). _(COSMETIC.)_

Самые отполированные экраны: онбординг (W06), семья (W05), auth. Самые срочные функциональные дыры: обработчик «новый тиждень» (W08) и недостижимость фидбека (W09).
